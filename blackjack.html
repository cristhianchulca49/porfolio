Blackjack API — Reactive Backend

100% reactive (non-blocking) backend service that allows playing Blackjack, managing statistics, and competing on a global leaderboard. Zero blocking threads thanks to Spring WebFlux and R2DBC.

Architecture and Technical Details

  • Hexagonal Architecture (Ports & Adapters) with three independent bounded contexts: game, player, and ranking.
  • The domain knows nothing of Spring, MongoDB, or any external dependencies — dependency arrows always point inward.
  • Polyglot persistence: MongoDB for game state and MySQL via R2DBC for profiles and ranking.
  • Reactive testing with WebTestClient and StepVerifier.
  • Dockerization and management via GitHub Projects and Kanban.
