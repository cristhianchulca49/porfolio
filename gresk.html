Final project (3 weeks)

GresK — Musical Event Platform

Designed from my experience as a musician, this tool connects artists with their audience through Spotify recommendations and capacity management. I combine the precision of Clean Architecture with a deep understanding of the musical environment to create scalable, user-oriented backend solutions.

Technical Features and Architecture

  • REST API development with Java and Spring Boot.
  • Authentication system with JWT and role-based access control (user, promoter, admin).
  • Ticket purchase implementation with concurrency control to prevent overbooking.
  • Integration with external APIs (Spotify, Cloudinary, Mapbox).
  • Persistence with PostgreSQL and migration management via Flyway.
  • Generation of unique QR codes for ticket validation.
  • Application of Domain-Driven Design (DDD) best practices and code organization.
